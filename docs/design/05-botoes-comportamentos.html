Caçada
Módulo 1 — scraping + extração
URL Alvo
https://exemplo.com
INICIAR
FURTIVO
INICIAR
idleem progresso → vira PAUSAR
Valida URL → inicia thread do scraper → Playwright abre página → extrai todos os assets (img, svg, ico) → salva em data/sessao_atual/ → posta progresso no log em tempo real.
FURTIVO
off = requests simpleson = Playwright com evasão
Toggle persistido no Grimório. Quando ativo: fingerprint rotation, jitter de delays, bypass Cloudflare, simulação de mouse. Custo: +2-5s por request.
PAUSAR
visível apenas durante scraping
Sinaliza thread para parar no próximo asset seguro → salva checkpoint em data/sessao_atual/.checkpoint.json → botão vira RETOMAR. Status: "sessão pausada".
CANCELAR
visível durante scraping ou pausado
Para thread imediatamente → não salva checkpoint → descarta sessão parcial → limpa data/sessao_atual/ → estado volta a idle.
Progresso: 12 baixados / 47 totais (25%)
ativo 12 / 47 (25%) VRAM: 0 GB sessão: nova
Córtex
Módulo 2 — visão IA + classificação
ANALISAR
habilitado após Caçada ter assets
Sobe Ollama em porta isolada → carrega Moondream → processa cada imagem em data/sessao_atual/ → salva tags + paleta em dataset_assets.csv → exibe grid imagem × JSON ao vivo.
EXPURGAR VRAM
habilitado quando Ollama está ativo
pkill no PID exato do Ollama → limpa OLLAMA_TMPDIR → aguarda confirmação de processo encerrado → atualiza VRAM: 0 GB no status. Emergency kill switch.
PAUSAR IA
durante análise ativa
Pausa fila de processamento → mantém Ollama vivo (VRAM ocupada) → botão vira RETOMAR IA. Útil para inspecionar resultados intermediários sem matar o processo.
K-MEANS
configurável no Grimório
Parâmetro para quantização de cores (default: 4 cores dominantes). Botão de ação rápida para reprocessar paleta de um asset selecionado sem rodar IA completa.
ollama ativo 8 / 12 analisados (66%) VRAM: 1.2 GB moondream: v2.1
Espólio
Módulo 3 — exportação de pacote
GERAR PACOTE
habilitado após análise IA completa
Agrupa assets em subpastas por categoria (icons/, backgrounds/, etc.) → gera dataset_assets.csv final → compacta tudo em output/beholder_YYYYMMDD_HHMMSS.zip → exibe caminho e tamanho.
ABRIR PASTA
sempre habilitado se output/ existe
Abre o diretório output/ no gerenciador de arquivos do sistema via xdg-open.
EXPORTAR CSV
separado do .zip
Exporta apenas o dataset_assets.csv para um caminho escolhido pelo usuário via diálogo de arquivo GTK.
LIMPAR SESSÃO
com confirmação obrigatória
Remove data/sessao_atual/ e o CSV intermediário → estado volta a zero. Diálogo de confirmação antes de executar.
12 assets prontos icons: 7 · bg: 3 · outros: 2 output: ~4.2 MB
Protocolo
Módulo 4 — batch + checkpoints
EXECUTAR LOTE
habilitado com ≥ 1 URL na fila
Processa URLs da fila em sequência → para cada URL: Caçada → Córtex → Espólio → salva checkpoint após cada URL concluída → ao final gera um .zip unificado.
ADICIONAR URL
sempre habilitado
Campo de texto inline → valida URL → adiciona à fila com status pendente. Suporta colar lista de URLs (uma por linha) de uma vez.
IMPORTAR LISTA
abre diálogo de arquivo
Carrega arquivo .txt com uma URL por linha → valida cada entrada → adiciona todas à fila de uma vez. Duplicatas são ignoradas automaticamente.
RETOMAR SESSÃO
visível quando checkpoint existe
Lê .checkpoint.json → restaura fila e posição → status: "sessão restaurada — continuando de 3/8" → retoma do próximo item pendente.
lote ativo 3 / 8 URLs (37%) checkpoint salvo próxima: site2.com
Grimório
Módulo 5 — configurações do sistema
SALVAR
habilitado quando há mudanças não salvas
Persiste configurações em ~/.config/beholder/config.ini via XDG standard (mesmo padrão do Conversor). Validação antes de salvar — campos inválidos destacados em vermelho.
TESTAR OLLAMA
verifica conexão com porta configurada
Faz ping na porta do Ollama → exibe latência e versão → confirma que Moondream está disponível. Feedback visual: verde (OK) / vermelho (falhou).
RESTAURAR PADRÕES
com confirmação
Reseta config.ini para valores de defaults.py → recarrega todos os campos do Grimório. Confirmação obrigatória antes de executar.
ABRIR LOGS
sempre habilitado
Abre logs/beholder.log no editor de texto padrão via xdg-open. Logs rotativos: 10 MB max, 3 backups.
